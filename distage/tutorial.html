    DIStage by example

    This tutorial will teach you how to build applications quickly and efficiently using DIStage! DIStage is a library that takes care of all the boring parts of wiring your app together, letting you focus on just the essentials.
    DIStage is staged, meaning that it’s split into several distinct stages, letting you add custom functionality between stages should you need it, yet it keeps a simple API.

    We’ll start with basic examples and by the end of this tutorial you’ll make a fully-fledged web application, complete with tests and configuration! So hang on tight, and let’s start with:

    Defining simple modules

    This is how Hello World looks in izumi-di

    import com.github.pshirshov.izumi.distage.Injectors
import com.github.pshirshov.izumi.distage.model.definition.ModuleDef

class Main {
  def helloWorld(): Unit =
    println("Hello World!")
}

object AppModules extends ModuleDef {
  make[Main]
}

object Main extends App {
  val injector = Injectors.bootstrap()
  val plan = injector.plan(AppModules)
  val app = injector.produce(plan)

  app.get[Main].helloWorld()
}

    That’s a lot to chew in one go, so let’s go line by line.

    object AppModules extends ModuleDef {
  make[Main]
}

    Here we define a Module for our application. A module is a set of classes we want to have in our application.
